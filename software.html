Software

WoD Server

The reference implementation of the WoD specification is an open source (MIT License), production ready, standalone server. It can be built from source (link coming soon) or run as a docker container. It implements the service API and server side semantics.

Get started now!

To start a WoD node server as a docker container run the following docker command:

      docker run --name wod-node1 -p 8888:8888 webofdata/node

The to test its running:

                curl http://localhost:8888/info
              
                curl http://localhost:8888/stores

WoD Client SDK

The client SDKs provide language specific libraries for communicating with a WoD service endpoint. These clients also provide implementations of the syncronisation features of the protocol.

The following client SDKs are available. See the individual project pages for how to install and use them.

  • SharpWoD - C# (dotnet core) client library implementation. v0.8
  • JsWoD - Javascript/Typescript client library implementation. v0.8
  • GoWoD - Go client library implementation - help needed
  • JWoD - Java client library implementation - help needed
  • PyWoD - Python client library implementation - help needed
